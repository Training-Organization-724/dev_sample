# Git 操作チートシート

Git（ギット）は、ファイルの変更履歴を記録・管理するためのバージョン管理システムです。
この資料では、覚えておくべき基本コマンドと操作の流れを解説します。

## ⚙️ 1. 事前準備（最初に1回だけ行う設定）

プロジェクトを始める前に、自分の名前とメールアドレスをGitに登録します。

### ステップ1：ユーザー名の設定
```bash
git config --global user.name "GitHubに登録したユーザ名"
※設定後に「git config --global user.name」を実行して設定済みを確認しましょう。
```

### ステップ2：メールアドレスの設定
```bash
git config --global user.email "GitHubに登録したメールアドレス"
※設定後に「git config --global user.email」を実行して設定済みを確認しましょう。
```

### ステップ3：SSH Keyの作成
```bash
ssh-keygen -t ed25519
```

### ステップ4：公開鍵をGitHubに登録
ローカルフォルダとGitHubのブラウザ上の操作が必要です。
1. C:\Users\student\\.sshフォルダを開く
2. pub拡張子が付いているファイルをメモ帳で開いて中身全てをコピペ
3. GitHub上で設定します。（テキストP.41~42を参照）

### ステップ5：ホスト認証鍵の取得
```bash
ssh git@github.com
```

## 🚀 3. 基本的な作業の流れ（日常のルーティン）

現状の確認や、最新の状態を取得することなど、<br>
ファイルの変更からリモートリポジトリ（GitHubなど）への反映までは、以下の6ステップが基本です。<br>
※初回の（リポジトリをGitHubで作成した）場合は[6. リポジトリ作成後の対応](#-6-リポジトリ作成後の対応)を参照してください。

### ステップ1：プロジェクトフォルダに移動する
```bash
cd C:\\pleiades\\2025-09\\workspace\\作成したプロジェクト名
```

### ステップ2：現状の確認
現在のブランチの状態や、変更されたファイルを確認します。
```bash
git status
```

### ステップ3：リモートから最新の状態を取得する

チーム開発などで、他の人が行った変更を自分のパソコンに取り込むコマンドです。
現状の確認後、編集中ファイル（modified）がある場合は競合に注意しましょう。

#### リモートの最新情報を取得して統合する
```bash
git pull origin main
```

>[!WARNING]
>Merge branch 'main' of github.com:Training-Organization-724/リポジトリ名<br>
>上記のメッセージが出力された場合はGit Bashで以下の対応を行ってください。

```bash
1. Escキーを押下
2. 「:wq!」をタイプ(Git Bash左下に出る)してEnterキーを押下（保存・終了コマンド）
3. 「git status」で自分が追加・変更しているファイル一覧を確認する
4. 一覧のファイルをeclipseで確認する
5. 確認して異常が出ていなければ再度ステップ3から実施し直す
※異常がある場合は修正してから再度ステップ3から実施し直す
※HTMLファイルの以上は分かりにくいので注意してください
```

### ステップ4：変更の登録（ステージング）
コミット（履歴の保存）をするために、対象のファイルを準備エリア（ステージ）に追加します。<br>
>[!WARNING]
>※git addで準備エリア（ステージ）に追加する前に必ずステップ2を行いましょう。<br>
>　エラーが発生し、その後のステップ5を正常に行えない場合があります。

- **特定のファイルを登録する場合**
  ```bash
  git add ファイル名
  ```
- **変更されたすべてのファイルを登録する場合**
  ```bash
  git add .
  ```

### ステップ5：履歴の保存（コミット）
ステージに追加した変更を、メッセージとともに履歴として保存します。
```bash
git commit -m "分かりやすい変更内容（例：ログイン画面を追加）"
```

### ステップ6：サーバーへアップロード（プッシュ）
ローカル（自分のパソコン）で行ったコミットを、リモートサーバー（GitHub）にアップロードします。
```bash
git push -u origin main
```
*(※ `main` の部分はブランチ名によって変わる場合があります)*

>[!WARNING]
>error、またはfatalが表示された場合は[4. お役立ちコマンド](#-4-お役立ちコマンド)の1~3を行い、ステップ3から再実行してください。<br>
>※修正したファイル内容が消えてしまう可能性があります。バックアップを取得しておきましょう。

## ⏪ 4. お役立ちコマンド

### 1. 直前のコミットを取り消す
```bash
git reset --soft HEAD~1
```

### 2. ステージしたファイルを戻す
```bash
git restore --staged ファイル名
```

### 3. ファイルの変更を直す（直前の状態に戻す）
```bash
git restore ファイル名
```

### 過去のコミット履歴を確認する
```bash
git log
```
※git logのオプションはこちらを参考に
[git log よく使うオプションまとめ](https://qiita.com/take4s5i/items/15d8648405f4e7ea3039)

## 🌿 5. ブランチ（Branch）の操作

ブランチを使うことで、メインの作業（`main`など）に影響を与えずに新機能の追加やバグ修正を並行して行うことができます。
※研修では使用しません。

### ブランチの一覧を確認する
```bash
git branch
```

### 新しいブランチを作成し、そのブランチに移動する
```bash
git checkout -b ブランチ名
```
*(※ Gitのバージョンが新しい場合は `git switch -c ブランチ名` でも可)*

### 既存のブランチに移動する
```bash
git checkout ブランチ名
```
*(※ Gitのバージョンが新しい場合は `git switch ブランチ名` でも可)*

### 他のブランチの変更を今のブランチに合流させる（マージ）
```bash
git merge 合流させたいブランチ名
```

## 🔔 6. リポジトリ作成後の対応

### ステップ1：GitHubで作成したリポジトリと同名のプロジェクトをeclipseで作成する

eclipseを使用してSpringプロジェクトを作成してください。<br>
DBを使用するため作成する時に依存関係を忘れずに追加してください。

### ステップ2：Git Bashを使用して以下のコマンドを実行する

- **1.フォルダに移動する**
  ```bash
  cd C:\\pleiades\\2025-09\\workspace\\作成したプロジェクト名
  ```

- **2.フォルダをGitの管理下にする**
  ```bash
  git init
  ```

- **3.フォルダ内ファイル全てをステージングに追加する**
  ```bash
  git add .
  ```

- **4.ステージングに追加したファイルをコミットする**
  ```bash
  git commit -m "first commit"
  ```

- **5.ローカルにmainブランチを作成する**
  ```bash
  git branch -M main
  ```

- **6.リモート(GitHub)にローカルの変更内容をアップロードするための準備**
  ```bash
  git remote add origin sshパス
  ※sshパスはGitHubからコピーしてください。
  ```

- **7.リモート(GitHub)にローカルの変更内容をアップロードする**
  ```bash
  git push -u origin main
  ```

## 💡 ワンポイントアドバイス
- 迷ったらまず **`git status`** を叩いて現状を確認しましょう。
- **`git add .`** や **`git commit`** をする前には、必ず **`git status`** で意図しないファイルが含まれていないか確認する癖をつけると安心です。
