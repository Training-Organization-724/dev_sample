# Git 初心者向けチートシート

Git（ギット）は、ファイルの変更履歴を記録・管理するためのバージョン管理システムです。
この資料では、初心者が最低限覚えておくべき基本コマンドと操作の流れを解説します。

## ⚙️ 1. 事前準備（最初に1回だけ行う設定）

プロジェクトを始める前に、自分の名前とメールアドレスをGitに登録します。

### ユーザー名の設定
```bash
git config --global user.name "あなたの名前"
```

### メールアドレスの設定
```bash
git config --global user.email "あなたのメールアドレス"
```

---

## 🚀 2. 基本的な作業の流れ（日常のルーティン）

ファイルの変更からリモートリポジトリ（GitHubなど）への反映までは、以下の4ステップが基本です。

### ステップ1：現状の確認
現在のブランチの状態や、変更されたファイルを確認します。
```bash
git status
```

### ステップ2：変更の登録（ステージング）
コミット（履歴の保存）をするために、対象のファイルを準備エリア（ステージ）に追加します。

- **特定のファイルを登録する場合**
  ```bash
  git add ファイル名
  ```
- **変更されたすべてのファイルを登録する場合**
  ```bash
  git add .
  ```

### ステップ3：履歴の保存（コミット）
ステージに追加した変更を、メッセージとともに履歴として保存します。
```bash
git commit -m "分かりやすい変更内容（例：ログイン画面を追加）"
```

### ステップ4：サーバーへアップロード（プッシュ）
ローカル（自分のパソコン）で行ったコミットを、リモートサーバー（GitHubなど）にアップロードします。
```bash
git push origin main
```
*(※ `main` の部分はブランチ名によって変わる場合があります)*

---

## 🌐 3. リモートから最新の状態を取得する

チーム開発などで、他の人が行った変更を自分のパソコンに取り込むコマンドです。

### リモートの最新情報を取得して統合する
```bash
git pull origin main
```

---

## 🌿 4. ブランチ（Branch）の操作

ブランチを使うことで、メインの作業（`main`など）に影響を与えずに新機能の追加やバグ修正を並行して行うことができます。

### ブランチの一覧を確認する
```bash
git branch
```

### 新しいブランチを作成し、そのブランチに移動する
```bash
git checkout -b ブランチ名
```
*(※ Gitのバージョンが新しい場合は `git switch -c ブランチ名` でも可)*

### 既存のブランチに移動する
```bash
git checkout ブランチ名
```
*(※ Gitのバージョンが新しい場合は `git switch ブランチ名` でも可)*

### 他のブランチの変更を今のブランチに合流させる（マージ）
```bash
git merge 合流させたいブランチ名
```

---

## ⏪ 5. お役立ちコマンド

### 過去のコミット履歴を確認する
```bash
git log
```

### 直前のコミットを取り消す
```bash
git reset --soft HEAD~1
```

### ファイルの変更を直す（直前の状態に戻す）
```bash
git restore ファイル名
```
