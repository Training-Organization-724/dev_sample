# Git 操作チートシート

Git（ギット）は、ファイルの変更履歴を記録・管理するためのバージョン管理システムです。
この資料では、覚えておくべき基本コマンドと操作の流れを解説します。

## ⚙️ 1. 事前準備（最初に1回だけ行う設定）

プロジェクトを始める前に、自分の名前とメールアドレスをGitに登録します。

### ステップ1：ユーザー名の設定
```bash
git config --global user.name "GitHubに登録したユーザ名"
※設定後に「git config --global user.name」を実行して設定済みを確認しましょう。
```

### ステップ2：メールアドレスの設定
```bash
git config --global user.email "GitHubに登録したメールアドレス"
※設定後に「git config --global user.email」を実行して設定済みを確認しましょう。
```

### ステップ3：SSH Keyの作成
```bash
ssh-keygen -t ed25519
```

### ステップ4：公開鍵をGitHubに登録
ローカルフォルダとGitHubのブラウザ上の操作が必要です。
1. C:\Users\student\\.sshフォルダを開く
2. pub拡張子が付いているファイルをメモ帳で開いて中身全てをコピペ
3. GitHub上で設定します。（テキストP.41~42を参照）

### ステップ5：ホスト認証鍵の取得
```bash
ssh git@github.com
```

## 🚀 3. 基本的な作業の流れ（日常のルーティン）

現状確認、最新の状態を取得することや、ファイルの変更からリモートリポジトリ（GitHubなど）への反映までは、以下の5ステップが基本です。

### ステップ1：現状の確認
現在のブランチの状態や、変更されたファイルを確認します。
```bash
git status
```

### ステップ2：リモートから最新の状態を取得する

チーム開発などで、他の人が行った変更を自分のパソコンに取り込むコマンドです。
現状の確認後、編集中ファイル（modified）がある場合は競合に注意しましょう。

#### リモートの最新情報を取得して統合する
```bash
git pull origin main
```

### ステップ3：変更の登録（ステージング）
コミット（履歴の保存）をするために、対象のファイルを準備エリア（ステージ）に追加します。

- **特定のファイルを登録する場合**
  ```bash
  git add ファイル名
  ```
- **変更されたすべてのファイルを登録する場合**
  ```bash
  git add .
  ```

### ステップ4：履歴の保存（コミット）
ステージに追加した変更を、メッセージとともに履歴として保存します。
```bash
git commit -m "分かりやすい変更内容（例：ログイン画面を追加）"
```

### ステップ5：サーバーへアップロード（プッシュ）
ローカル（自分のパソコン）で行ったコミットを、リモートサーバー（GitHub）にアップロードします。
```bash
git push -u origin main
```
*(※ `main` の部分はブランチ名によって変わる場合があります)*

## 🌿 4. ブランチ（Branch）の操作

ブランチを使うことで、メインの作業（`main`など）に影響を与えずに新機能の追加やバグ修正を並行して行うことができます。
※研修では使用しません。

### ブランチの一覧を確認する
```bash
git branch
```

### 新しいブランチを作成し、そのブランチに移動する
```bash
git checkout -b ブランチ名
```
*(※ Gitのバージョンが新しい場合は `git switch -c ブランチ名` でも可)*

### 既存のブランチに移動する
```bash
git checkout ブランチ名
```
*(※ Gitのバージョンが新しい場合は `git switch ブランチ名` でも可)*

### 他のブランチの変更を今のブランチに合流させる（マージ）
```bash
git merge 合流させたいブランチ名
```

---

## ⏪ 5. お役立ちコマンド

### 過去のコミット履歴を確認する
```bash
git log
```

### 直前のコミットを取り消す
```bash
git reset --soft HEAD~1
```

### ファイルの変更を直す（直前の状態に戻す）
```bash
git restore ファイル名
```

## 💡 ワンポイントアドバイス
- 迷ったらまず **`git status`** を叩いて現状を確認しましょう。
- **`git add .`** や **`git commit`** をする前には、必ず **`git status`** で意図しないファイルが含まれていないか確認する癖をつけると安心です。
